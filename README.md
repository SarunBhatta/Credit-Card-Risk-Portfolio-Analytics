# Credit Card Risk & Portfolio Analytics Dashboard

## Project Overview

This project analyzes credit card customer data to understand default
risk, payment behavior, and overall portfolio characteristics. The
dashboard was built in Tableau to provide interactive insights into
customer credit risk and repayment patterns.

## Dataset

The project uses the UCI Credit Card Default dataset, which contains
information on 30,000 credit card customers in Taiwan, including: -
Credit limits - Customer demographics - Repayment status history - Bill
statements - Previous payments - Default status for the following month

## Dashboard Features

### KPI Cards

-   Default Rate
-   Average Credit Limit
-   Total Customers

### Visualizations

-   Credit Limit Distribution
-   Default Risk by Payment Status
-   Customer Distribution by Payment Status
-   Default Rate by Credit Limit

### Interactive Filters

-   Age Group
-   Gender
-   Payment Status

## Key Insights

-   The overall default rate is approximately 22%.
-   Customers with payment delays are significantly more likely to
    default.
-   Customers with payment delays of two or more months represent a
    high-risk segment.
-   Higher credit limit customers generally demonstrate lower default
    rates.
-   Most customers fall within lower to mid-range credit limit
    categories.

## Project Summary

This project demonstrates how data visualization can be used to identify
customer risk patterns and support data-driven decision-making. Through
interactive dashboards and business-focused KPIs, users can explore
customer segments, monitor default risk, and better understand portfolio
performance.
